## 5. مشکلات Rate Limit و کندی

### ۵.۱ ⚠️ Google Translate — **بالاترین ریسک**

| مشکل | جزئیات |
|-------|-------|
| **بدون API Key** | از `translate.googleapis.com/translate_a/single?client=gtx` استفاده می‌کند — این endpoint غیررسمی است |
| **Rate Limit ناشناخته** | Google هیچ SLA رسمی برای این endpoint ندارد. معمولاً پس از ~50-100 درخواست در دقیقه، **503 یا 429** برمی‌گرداند |
| **ترجمه Sequential** | `for...of` روی ۶ آیتم، هرکدام ۲ ترجمه (title + desc) = ۱۲ درخواست پشت سر هم. اگر یکی شکست بخورد، متن اصلی انگلیسی نمایش داده می‌شود |
| **بدون Retry** | اگر ترجمه شکست بخورد، فقط `catch` می‌شود و متن اصلی برگردانده می‌شود (L1532) |
| **تأثیر روی Latency** | هر ترجمه ~200-800ms. ۱۲ ترجمه = **2.4-9.6 ثانیه** فقط برای ترجمه |
| **دو لایه ترجمه:** | Backend ترجمه می‌کند + اگر Fallback فعال شود، Frontend هم دوباره ترجمه می‌کند! |
